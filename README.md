# LOT-IMS — 구글 클라우드 통합 재고관리 시스템

로트번호(`품목코드-제조일자-일련번호`) 바코드로 입·출고/재고/위치를 관리하고,
**문서·사진 보관(Drive) · 품질 이상신고 · 자동 알림(Chat/Gmail) · 리포트 대시보드**까지
구글 클라우드만으로 운영하는 무료 재고관리 시스템입니다. 서버가 필요 없습니다.

```
휴대폰/PC 브라우저 ── GitHub Pages (index.html)
        │  HTTPS (fetch)
        ▼
Google Apps Script 웹앱 (Code.gs)  ← 인증·재고검증·잠금(LockService)
        │
        ├──▶ Google Sheets   (Users/Items/Lots/Locations/History/Settings/Documents/Issues)
        ├──▶ Google Drive    (검수사진·거래명세서·이상신고 사진)
        ├──▶ Google Chat     (안전재고 미달·중대 이상신고 실시간 알림)
        ├──▶ Gmail           (일일 재고 다이제스트 메일)
        └──▶ Looker Studio   (Report_Stock / Report_Tx 시트를 그대로 연결)
```

## 저장소 구성

```
your-repo/
├── index.html          # 페이지 뼈대 (HTML만, 가볍게 유지)
├── css/
│   └── app.css         # 전체 스타일
├── js/
│   ├── app-core.js       # 상태관리 · API 클라이언트 · 로그인 · 재고현황/위치/이력
│   ├── app-operations.js # 바코드 생성 · 스캔 입출고 · 관리(사용자/품목/위치/데이터)
│   └── app-extra.js      # 문서함 · 품질신고 · 리포트 · 알림설정 · 개발메모
├── Code.gs              # 백엔드 API — 구글 시트의 Apps Script에 붙여넣기
└── schema.sql           # (참고) 동일 구조의 PostgreSQL 스키마 — 추후 DB 이관용
```

세 JS 파일은 반드시 **`app-core.js` → `app-operations.js` → `app-extra.js`** 순서로 로드되어야 합니다
(뒤 파일이 앞 파일에서 정의한 함수를 사용하기 때문). `index.html`에 이미 이 순서로 `<script src>`가
들어있으니, 파일을 추가하거나 순서를 바꾸지만 않으면 됩니다.

> 이전에는 HTML/CSS/JS가 파일 하나(`index.html`, 약 80KB)에 모두 들어있었는데,
> 기능별로 파일을 나눠 필요한 부분만 찾아 수정하기 쉽고, 브라우저가 CSS/JS를 별도로
> 캐싱해 다음 방문 때 더 빠르게 뜨도록 구조를 정리했습니다.

## 전체 기능

- **바코드 생성**: 로트번호 자동 채번, QR/Code128, 라벨 PNG/인쇄
- **스캔 입·출고·이동·조정**: 카메라·리더기 스캔으로 **입고 / 출고 / 위치이동(MOVE) / 재고실사조정(ADJUST)** 처리, 수량/일시/담당자/사유 자동 기록
- **재고 현황 / 재고 위치**: 안전재고 게이지, 창고/구역/랙 조회, **재고·이력 CSV 내보내기**(엑셀 호환)
- **다크/라이트 모드**: 헤더 🌙/☀️ 토글, 브라우저에 선택 저장(기본은 OS 설정 따름)
- **문서함**: 로트별 검수사진·거래명세서·인증서를 구글 드라이브에 보관, 앱에서 썸네일로 열람
- **품질검사/이상신고**: 경미·중대·긴급 등급 신고, 사진 첨부, 상태 관리(접수→처리중→완료)
- **자동 알림**: 안전재고 미달·중대 신고 시 **Google Chat 실시간 알림**, 매일 **Gmail 다이제스트**
- **리포트/대시보드**: 인앱 재고·추이 차트, **Looker Studio** 연동용 평탄화 시트 자동 유지
- **개발 메모(개발로그)**: 개발 방향·결정사항·할 일·버그를 타임라인으로 기록 (관리 → 메모)
- **이력**: 전 트랜잭션 전수 기록(구글 시트 History 탭에 영구 보관)

## 설치 (총 15분)

### 1단계 — 구글 시트 + Apps Script 백엔드

1. [sheets.new](https://sheets.new) 에서 새 구글 시트 생성 (이름 예: `LOT-IMS-DB`)
2. 메뉴 **확장 프로그램 → Apps Script** 클릭
3. 기본 코드를 지우고 이 저장소의 **`Code.gs` 전체를 붙여넣기** → 저장(💾)
4. 상단 함수 선택 드롭다운에서 **`setup`** 선택 → **실행**
   - 권한 승인 창: *권한 검토 → 계정 선택 → 고급 → (프로젝트명)(안전하지 않음)으로 이동 → 허용*
   - 완료되면 시트에 **Users/Items/Lots/Locations/History/Settings/Documents/Issues** 탭,
     관리자 계정(`admin / admin1234`), 그리고 구글 드라이브에 **`LOT-IMS-Files`** 폴더가 자동 생성됩니다
5. **배포 → 새 배포 → 유형(⚙️) → 웹 앱**
   - **실행 계정: 나** / **액세스 권한: 모든 사용자** → **배포**
   - 발급된 **웹 앱 URL**(`https://script.google.com/macros/s/…/exec`) 복사

> 🔁 이후 `Code.gs`를 수정하면 **배포 → 배포 관리 → ✏️ → 새 버전**으로 재배포해야 반영됩니다.

### 2단계 — GitHub Pages 프론트엔드

1. 이 저장소를 Fork 하거나, 새 저장소에 `index.html` / `css/app.css` / `js/app-core.js` /
   `js/app-operations.js` / `js/app-extra.js` 를 **폴더 구조를 그대로 유지**해서 업로드
   (드래그 앤 드롭 업로드 시 `css`, `js` 폴더가 각각 만들어지는지 확인하세요)
2. (선택) `js/app-core.js` 상단 `DEFAULT_API_URL`에 1단계 웹 앱 URL을 넣으면 로그인 화면에서 주소 입력이 생략됩니다
3. 저장소 **Settings → Pages → Branch: main / (root) → Save**
4. 1~2분 후 `https://<계정명>.github.io/<저장소명>/` 접속 (GitHub Pages는 기본 HTTPS라 카메라 스캔도 바로 동작합니다)

### 3단계 — 첫 로그인

| 항목 | 값 |
|---|---|
| 서버 주소 | 1단계에서 복사한 웹 앱 URL |
| 아이디 / 비밀번호 | `admin` / `admin1234` |

로그인 후 **관리 → 사용자**에서 비밀번호를 변경하고 작업자 계정을 추가하세요.

## 4단계 — 구글 연동 기능 설정 (선택이지만 권장)

앱 하단 **관리 → 알림** 탭에서 아래 항목을 설정합니다.

### 💬 Google Chat 실시간 알림
1. Google Chat에서 알림을 받을 스페이스(채팅방) 열기
2. 스페이스 이름 옆 ▾ → **앱 및 통합 관리 → 웹훅 추가**
3. 이름(예: `LOT-IMS`) 입력 후 생성된 **웹훅 URL** 복사
4. 앱의 **관리 → 알림 → Webhook URL**에 붙여넣고 저장 → **테스트 발송**으로 확인

이후 다음 상황에서 실시간으로 채팅 알림이 옵니다.
- 출고 처리로 재고가 안전재고 아래로 **새로 진입**했을 때
- **중대/긴급** 등급의 품질 이상신고가 접수되었을 때
- 일일 다이제스트 발송 시 요약 메시지

### 📧 Gmail 일일 다이제스트
1. 앱의 **관리 → 알림 → 수신 이메일**에 알림받을 주소를 쉼표로 구분해 입력
2. 유통기한 임박 기준일수, 발송 시각을 설정 후 **저장**
3. **일일 알림 자동 발송 설치** 버튼 클릭 → 매일 지정 시각에 Apps Script 트리거로 자동 실행
4. **지금 테스트 발송**으로 즉시 확인 가능

> Gmail 발송은 Apps Script의 `MailApp` 할당량(개인 Google 계정 기준 1일 100통)을 사용합니다.
> Google Workspace 계정은 더 높은 한도(1일 1,500통)가 적용됩니다.

### 🗂️ 문서/사진 보관 (Google Drive)
- 별도 설정 없이 `setup()` 실행 시 생성된 **`LOT-IMS-Files`** 폴더에 자동 저장됩니다
- 앱의 **더보기 → 문서함**에서 로트번호로 검색 후 사진/PDF 업로드·열람·삭제
- 스캔 화면에서 로트를 스캔하면 **📁 문서/사진** 버튼으로 바로 이동 가능
- 이미지는 브라우저에서 최대 1280px·JPEG 72% 품질로 자동 압축 후 업로드되어 용량과 속도를 아낍니다
- 업로드된 파일은 "링크가 있는 모든 사용자 열람 가능"으로 공유되어 앱 내 썸네일이 표시됩니다.
  민감한 문서를 다룬다면 이 공유 범위를 조직 정책에 맞게 조정하세요(Drive에서 파일별로 변경 가능)

### 🚨 품질검사 / 이상신고
- **더보기 → 품질신고**에서 로트 지정(선택) + 심각도(경미/중대/긴급) + 사진 첨부로 신고
- 목록 탭에서 상태를 접수→처리중→완료로 변경, 처리 메모 기록 가능
- 미해결 건수는 하단 **더보기** 탭에 빨간 배지로 표시됩니다

### 📊 리포트 / Looker Studio 연동
- **더보기 → 리포트**: 품목별 재고, 최근 14일 입출고 추이, 안전재고/이상신고 건수를 인앱 차트로 확인
- 더 깊은 분석이 필요하면 연결된 구글 시트의 **`Report_Stock`**, **`Report_Tx`** 탭(품목명·창고가
  이미 조인되어 있음)을 [Looker Studio](https://lookerstudio.google.com)의 **Google Sheets 커넥터**로
  바로 연결하세요. 이 두 탭은 입출고·로트 생성 시마다 자동으로 최신 상태로 재생성됩니다.

## 사용 흐름 요약

1. **관리(관리자)** — 품목/위치/사용자 등록, 알림(Chat/Gmail) 설정, 개발 메모 기록
2. **바코드 생성** — 로트 자동 채번 → QR/Code128 라벨 다운로드·인쇄
3. **스캔 입·출고** — 카메라/리더기 스캔 → 입고·출고·위치이동, FIFO 경고, 문서함/이상신고 바로가기
4. **재고 현황 / 재고 위치** — 실시간 재고와 보관 위치 확인
5. **더보기 → 문서함 / 품질신고 / 리포트** — 문서 보관, 이상신고 처리, 대시보드 확인
6. **관리 → 메모** — 개발 방향·결정 이유·할 일·버그를 시간순으로 기록 (팀 개발로그)
7. **이력** — 최근 500건 인앱 조회(전체는 시트 History 탭)

**바코드 리더기(USB/블루투스)**: 키보드 입력 방식이므로 스캔 탭의 수동 입력칸에 커서를 두고 스캔하면 됩니다.

## 데이터 정합성 · 보안

- 모든 쓰기(입출고/로트생성/문서·이슈 등록/마스터 변경)는 Apps Script **LockService**로
  직렬화되어 다중 사용자 동시 처리에도 재고 수량이 어긋나지 않습니다.
- 재고 검증(부족 출고 차단), 로트 중복 방지는 **서버에서** 수행됩니다.
- 비밀번호는 SHA-256 해시로 저장되며, 모든 요청은 인증을 거칩니다.
- History 시트는 append-only로 사용하세요(수동 편집 금지 권장).
- 구글 드라이브 **버전 기록**으로 시트를, **휴지통**으로 삭제된 문서를 복원할 수 있습니다.

### 알아둘 한계
- 웹 앱 URL을 아는 사람은 로그인 시도를 할 수 있으므로 **비밀번호를 충분히 길게** 설정하세요.
- 업로드된 문서/사진은 "링크 소유자 열람 가능"으로 공유됩니다 — 민감 문서는 공유 범위를 재검토하세요.
- Apps Script 무료 할당량(실행 횟수/시간, 메일 발송 수)은 소규모 팀에는 충분하지만,
  대규모 트래픽이 필요해지면 `schema.sql` 기반의 전용 DB/API 서버로 이관을 권장합니다.

## 로트번호 / 바코드 규칙

```
로트번호 = 품목코드(영문·숫자 2~12자) - 제조일자(YYYYMMDD) - 일련번호(001~)
예)       BRKT01-20260714-001
바코드    QR(권장, 휴대폰 인식률 높음) 또는 Code128 1D
```
바코드에는 로트번호 문자열만 인코딩하므로 시중의 모든 1D/2D 리더기와 호환됩니다.
